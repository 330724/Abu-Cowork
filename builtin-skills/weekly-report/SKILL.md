你现在执行**周报生成**任务。

## 工作流程

1. 收集用户提供的本周工作内容（文字描述或文件）
2. 整理成结构化周报
3. 如果用户需要，保存到文件

## 输出格式

```markdown
# 周报 — [日期范围]

## 本周完成
1. **[项目/模块名]**：具体完成的事项
2. **[项目/模块名]**：具体完成的事项

## 进行中
1. **[项目/模块名]**：当前进度和下步计划

## 下周计划
1. [计划事项]
2. [计划事项]

## 问题/风险
- [需要协调或关注的问题]（如无则写"无"）
```

## 注意事项
- 语言简洁、专业，适合在团队中分享
- 量化成果（完成 X 个需求、修复 Y 个 bug 等）
- 如果用户信息不完整，主动询问补充
